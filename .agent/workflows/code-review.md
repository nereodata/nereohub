# Code Review Workflow

1.  **Identificar Ámbito**: Determinar qué archivos o bloques de código deben ser revisados.
2.  **Ejecutar Skill**: Usar el skill `code-reviewer` para analizar la seguridad, performance, testeabilidad y cumplimiento de requisitos.
3.  **Generar Reporte**: Documentar los hallazgos en la carpeta `docs/review/code_reviews/`.
4.  **Veredicto**: Informar al usuario si el código es apto para continuar o requiere correcciones.

# Documentation Workflow

1.  **Analizar Cambios**: Identificar el impacto en la arquitectura o diseño del sistema.
2.  **Ejecutar Skill**: Usar `doc-generator` para actualizar o crear los archivos `.md` en `docs/architecture/`.
3.  **Compliance Check**: Si hay cambios en seguridad, asegurar que la documentación en `docs/architecture/security/` sea actualizada.
4.  **Confirmar**: Verificar que la documentación es fiel a la implementación actual.

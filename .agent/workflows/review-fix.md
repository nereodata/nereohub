# Review Fix Workflow

1.  **Analizar el Fix**: Identificar qué issues del review anterior o anomalías se están resolviendo.
2.  **Ejecutar Skill**: Usar `bug-fix-reviewer` para verificar que la solución es sólida y no tiene regresiones.
3.  **Generar Reporte**: Documentar en `docs/review/code_reviews/` con el formato de re-review.
4.  **Aprobar/Rechazar**: Determinar si la corrección es suficiente para cerrar la anomalía.
